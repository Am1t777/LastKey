Why does this exist?

😢 What happens to your digital life when you're gone?

  • 1 Your bank passwords, crypto keys, and important documents are locked away in your head — no one can access them after you're gone.
  • 2 Storing passwords in a shared note or email is insecure — anyone who sees it now has full access.
  • 3 You need a system that waits until the right moment — only releasing secrets after it's truly necessary.

Zero-knowledge security

🔐 How Encryption Works

1
A random AES-256 key is generated uniquely for each secret you store.
2
Your secret content is encrypted with that AES key using AES-256-GCM (authenticated encryption).
3
The AES key itself is encrypted using a key derived from your password + Argon2id — stored as owner_encrypted_key.
4
For each beneficiary, the AES key is re-encrypted with their RSA-2048 public key — stored in secret_assignments.
5
To read a secret: your password unlocks the AES key → AES key decrypts the content. Server never touches plaintext.

Security features

🛡️ Security Layers

🚫

Rate Limiting

5 attempts/min on auth. 10/min on verifier confirm/deny. Powered by slowapi.

🌐

CORS

Only the configured frontend origin can make cross-origin API requests.

🪙

CSRF Protection

All state-changing requests must include X-Requested-With header.

🔒

Security Headers

HSTS, X-Frame-Options: DENY, CSP, X-Content-Type-Options, Referrer-Policy.

📋

Audit Logging

Every sensitive action logged with user ID, action name, IP, and timestamp.

🔑

Zero Knowledge

Server stores only ciphertext. Passwords and plaintext never leave the browser.

End-to-end user journey

🚀 Full User Flow

  • 1 Register & log in. JWT token is issued. Check-in timer starts.
  • 2 Add beneficiaries. Generate an RSA key pair per beneficiary. Give them the private key.
  • 3 Store secrets. Enter title + content + your password. Content is AES-encrypted in the browser before upload.
  • 4 Assign secrets. Enter your password again to re-encrypt the AES key for each beneficiary's RSA key.
  • 5 Set a trusted verifier. Add the name + email of someone you trust to verify your status.
  • 6 Keep checking in. Click the link in the reminder email every 30 days (or your chosen interval).
  • 7 If the switch fires: verifier clicks confirm → beneficiaries get an email with a personal link → they paste their private key → secrets decrypted in browser.
